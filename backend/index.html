Overview

Pengantar

Tabunganku API adalah RESTful backend untuk aplikasi pencatatan tabungan. Mendukung dua mode: tabungan pribadi (per user) dan tabungan bersama (kolaboratif, berbasis grup dengan sistem invite code).

Authentication

Skema Autentikasi

Bearer Token (Supabase JWT)
Semua endpoint yang memerlukan autentikasi harus menyertakan header Authorization: Bearer <access_token>. Token diperoleh dari endpoint POST /api/auth/login. Token diverifikasi langsung ke Supabase Auth menggunakan supabase.auth.getUser(token).

Token expired dikembalikan dengan status 401. Gunakan POST /api/auth/refresh dengan refresh_token untuk mendapatkan access token baru tanpa login ulang.
Header yang digunakan
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Conventions

Format Respons

Seluruh endpoint mengembalikan struktur JSON yang konsisten dengan field success, message, dan data.

Success Response 200 / 201
{
  "success": true,
  "message": "Pesan deskriptif dalam Bahasa Indonesia",
  "data": {
    // payload sesuai endpoint
  }
}
Error Response 400 / 401 / 403 / 404 / 500
{
  "success": false,
  "message": "Deskripsi error",
  "errors": null  // atau array Zod errors saat validasi gagal
}
Validation Error (Zod) 400
{
  "success": false,
  "message": "Validation error",
  "errors": [
    { "field": "email", "message": "Format email tidak valid." },
    { "field": "password", "message": "Password minimal 8 karakter." }
  ]
}
Conventions

HTTP Status Codes

StatusArti
200Sukses — data berhasil diambil atau diproses
201Created — resource baru berhasil dibuat
400Bad Request — validasi gagal, input salah, atau saldo tidak mencukupi
401Unauthorized — token tidak ada, format salah, atau expired
403Forbidden — user bukan anggota grup atau bukan owner
404Not Found — resource tidak ditemukan
500Internal Server Error — error dari Supabase atau server
Conventions

Pagination & Filtering

Sistem pagination berbasis offset. Query param page (default: 1) dan limit (default bervariasi per endpoint, max 100) mengontrol jumlah data yang dikembalikan. Data selalu diurutkan berdasarkan created_at dengan arah yang dikontrol param sort.
ParamTipeDefaultMaxKeterangan
pageinteger1—Nomor halaman, minimum 1
limitinteger10 / 20100Savings: default 10 · Transactions & Shared: default 20
sortenumdesc—Urutan berdasarkan created_at. Nilai: asc atau desc
searchstring——Pencarian case-insensitive (ilike) pada nama/deskripsi
Struktur pagination dalam respons
{
  "pagination": {
    "total": 42,         // total record keseluruhan
    "page": 1,          // halaman sekarang
    "limit": 10,         // jumlah per halaman
    "totalPages": 5    // total halaman (min 1)
  }
}

Auth Endpoints

Manajemen sesi pengguna berbasis Supabase Auth. Menggunakan email/password dengan JWT access token + refresh token.

POST
/api/auth/register
Registrasi akun baru

Request Body

FieldTipeWajibAturan
emailstring✓ WajibFormat email valid
passwordstring✓ WajibMinimal 8 karakter

Contoh Request

application/json
{

  "password": "rahasia123"
}

Contoh Respons

201 Created
{
  "success": true,

  "data": {

  }
}

Kemungkinan Respons

201
Registrasi berhasil. Email verifikasi dikirim oleh Supabase.
400
Validasi gagal (email tidak valid / password terlalu pendek) atau email sudah terdaftar di Supabase.
POST
/api/auth/login
Login dan dapatkan access token

Request Body

FieldTipeWajibAturan

passwordstring✓ WajibTidak boleh kosong

Contoh Respons

  "message": "Login sukses!",

    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",        // access_token

      "id": "a1b2c3d4-...",

Login berhasil. Simpan token (access token) dan refresh_token di client.

Email atau password salah.

Logout berhasil. Hapus token dari client.
401
Token tidak ditemukan atau tidak valid.

Transactions

Transaction Endpoints

Pencatatan transaksi (setoran & penarikan) untuk tabungan pribadi. Setiap transaksi secara otomatis memperbarui current_amount di tabel savings_goals. Tabel: transactions.

Saldo otomatis: Saat transaksi dibuat, dihapus, atau diubah, saldo (current_amount) pada savings goal terkait otomatis disesuaikan. Penarikan yang melebihi saldo akan ditolak dengan error 400.

/api/transactions
Catat transaksi baru (deposit atau withdrawal)
🔒 Requires Auth

Request Body

FieldTipeWajibAturan

amountnumber✓ WajibHarus positif (> 0)

application/json

}

{

  "message": "Transaksi berhasil dicatat.",
  "data": {

      "id": "txn-uuid-...",

      "savings_id": "550e8400-...",

Dashboard

Dashboard Endpoints

Endpoint agregasi data untuk tampilan ringkasan pengguna. Semua kalkulasi dilakukan di server menggunakan data dari savings_goals dan transactions.

GET
/api/dashboard
Ringkasan total saldo, progress, dan statistik transaksi
🔒 Requires Auth

Contoh Respons

200 OK
{
  "success": true,
  "message": "Dashboard berhasil diambil.",
  "data": {
    "dashboard": {
      "total_savings": 3500000,        // total saldo dari semua savings
      "total_target": 10000000,        // total target dari semua savings
      "progress": 35,                  // persentase pencapaian (0–100)
      "completion_rate": 35,            // sama dengan progress
      "remaining_target": 6500000,     // sisa yang harus ditabung (min 0)
      "total_transactions": 24,         // total jumlah transaksi
      "total_deposit": 5000000,         // total semua setoran
      "total_withdrawal": 1500000,      // total semua penarikan
      "num_savings_goals": 3            // jumlah target tabungan aktif
    }
  }
}
Formula: progress = Math.round((total_savings / total_target) * 100). Jika tidak ada savings goal, progress bernilai 0.

Shared Savings

Shared Savings Endpoints

Tabungan kolaboratif yang bisa diikuti oleh banyak user. Setiap grup memiliki satu owner dan banyak member. Bergabung menggunakan invite code 8 karakter (hex uppercase). Tabel: shared_savings, shared_members.

Sistem Peran (Role):
owner — Pembuat grup. Memiliki akses penuh termasuk update dan delete grup.
member — User yang bergabung via invite code. Bisa melihat dan melakukan transaksi, tapi tidak bisa update/delete grup.
POST
/api/shared-savings
Buat grup tabungan bersama baru
🔒 Requires Auth

Request Body

FieldTipeWajibAturan
namestring✓ WajibMin 1 karakter (setelah trim)
target_amountnumber✓ WajibHarus positif (> 0). Bisa angka atau string angka (coerce)
descriptionstringOpsionalMaks 255 karakter. Default "" jika tidak diisi.

Contoh Respons

201 Created
{
  "success": true,
  "message": "Tabungan bersama berhasil dibuat.",
  "data": {
    "shared_savings": {
      "id": "ss-uuid-...",
      "owner_id": "a1b2c3d4-...",
      "name": "Dana Reuni SMA",
      "description": "Untuk acara reuni 10 tahun",
      "target_amount": 10000000,
      "current_amount": 0,
      "invite_code": "A3F8C21B",  // 8 karakter hex uppercase
      "status": "active",
      "created_at": "2025-01-01T00:00:00Z"
    },
    "member": {
      "id": "member-uuid-...",
      "shared_savings_id": "ss-uuid-...",
      "user_id": "a1b2c3d4-...",
      "role": "owner"
    }
  }
}
Invite Code: Dibuat otomatis menggunakan crypto.randomBytes(4).toString('hex').toUpperCase(). Unik dijamin dengan retry hingga 5 kali jika ada collision. Bagikan kode ini ke anggota yang ingin bergabung.

Shared Transactions

Shared Transaction Endpoints

Pencatatan transaksi untuk grup tabungan bersama. Semua anggota (owner & member) bisa membuat dan mengelola transaksi. Saldo grup otomatis diperbarui. Tabel: shared_transactions.

POST
/api/shared-transactions
Catat transaksi baru untuk grup tabungan bersama
🔒 Requires Auth

Request Body

FieldTipeWajibAturan
shared_savings_idUUID✓ WajibID grup yang valid, user harus menjadi anggota
typeenum✓ WajibNilai: deposit atau withdrawal
amountnumber✓ WajibHarus positif (> 0). Bisa string angka (coerce)
descriptionstringOpsionalMaks 255 karakter. Default "".

Contoh Respons

201 Created
{
  "success": true,
  "message": "Transaksi tabungan bersama berhasil dicatat.",
  "data": {
    "shared_transaction": {
      "id": "st-uuid-...",
      "shared_savings_id": "ss-uuid-...",
      "user_id": "a1b2c3d4-...",  // user yang melakukan transaksi
      "type": "deposit",
      "amount": 500000,
      "description": "Iuran bulan Januari",
      "created_at": "2025-01-10T09:00:00Z"
    },
    "saldo_sekarang": 3500000
  }
}

Kemungkinan Respons

201
Transaksi berhasil dan saldo grup diperbarui.
400
Validasi gagal atau saldo grup tidak mencukupi untuk withdrawal.
403
User bukan anggota grup.
404
Grup tidak ditemukan.
PATCH
/api/shared-transactions/:id
Update transaksi bersama
🔒 Requires Auth
Verifikasi bahwa user adalah anggota grup (bukan hanya pemilik transaksi). Mekanisme rollback dan reapply saldo sama seperti endpoint PATCH /transactions/:id.

Request Body (semua opsional, min 1)

FieldTipeAturan
typedeposit | withdrawalUbah jenis transaksi
amountnumberHarus positif (> 0)
descriptionstringMaks 255 karakter

Kemungkinan Respons

200
Transaksi berhasil diperbarui. Respons berisi shared_transaction dan saldo_sekarang.
400
Saldo tidak mencukupi atau tidak ada field yang dikirim.
403
User bukan anggota grup transaksi ini.
404
Transaksi tidak ditemukan.
DELETE
/api/shared-transactions/:id
Hapus transaksi bersama dan kembalikan saldo
🔒 Requires Auth
Saldo grup otomatis di-rollback saat transaksi dihapus. User harus menjadi anggota grup, tapi tidak harus pemilik transaksi.

Contoh Respons

200 OK
{
  "success": true,
  "message": "Transaksi tabungan bersama berhasil dihapus.",
  "data": {
    "saldo_sekarang": 3000000
  }
}

Kemungkinan Respons

200
Transaksi dihapus, saldo grup di-rollback.
403
User bukan anggota grup transaksi ini.
404
Transaksi tidak ditemukan.
